# Architecture

## Purpose

This directory stores generated and maintained knowledge derived from raw sources, questions, and synthesis work.

## Classification Principle

The wiki starts with a minimal overview. Domain-specific categories should emerge from actual source material and recurring queries.

## Direct Children

This directory currently has no child directories.

| Child | Definition | Boundary | Excludes |
|---|---|---|---|

## Progressive Disclosure

`overview.md` gives the broadest reading entry point. Future child directories should expose increasingly specific knowledge areas, with each directory explaining its own classification boundary.

## Change Rules

Create a child directory only when multiple wiki pages share a stable conceptual, organizational, or project boundary.

Do not create speculative taxonomies before there is enough content to justify them.
